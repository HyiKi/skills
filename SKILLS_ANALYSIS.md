# Skills 目录分析总结

本文档对 `skills/` 目录下的每个 skill 进行了详细分析，按点归纳总结其作用和功能。

## 1. algorithmic-art (算法艺术)

**作用**: 使用 p5.js 创建算法生成艺术，支持种子随机数和交互式参数探索

**核心功能**:

- 创建算法哲学（Algorithmic Philosophy）：生成 4-6 段的设计理念文档
- 实现 p5.js 生成艺术：将哲学转化为可交互的 HTML 艺术作品
- 种子随机数：使用 Art Blocks 模式确保可重现性
- 参数控制：提供实时可调的参数（粒子数量、噪声尺度、速度等）
- 种子导航：支持前一个/下一个种子、随机种子、跳转到指定种子
- 单文件输出：生成自包含的 HTML 文件，包含所有代码和 UI

**技术特点**:

- 使用 p5.js 库进行图形生成
- 支持流程场、粒子系统、噪声场等算法
- 强调算法表达而非静态图像
- 提供模板文件（viewer.html）作为基础结构

## 2. brand-guidelines (品牌指南)

**作用**: 将 Anthropic 官方品牌颜色和字体应用到任何需要品牌风格的工件上

**核心功能**:

- 品牌颜色应用：
  - 主色：深色 (#141413)、浅色 (#faf9f5)、中灰 (#b0aea5)、浅灰 (#e8e6dc)
  - 强调色：橙色 (#d97757)、蓝色 (#6a9bcc)、绿色 (#788c5d)
- 字体应用：
  - 标题（24pt+）：Poppins 字体
  - 正文：Lora 字体
  - 自动回退到 Arial/Georgia
- 智能样式应用：根据背景自动选择文本颜色
- 形状和强调色：非文本形状使用强调色循环

**使用场景**: 需要应用公司品牌风格的可视化内容

## 3. canvas-design (画布设计)

**作用**: 使用设计哲学创建精美的视觉艺术作品（.png 和 .pdf 格式）

**核心功能**:

- 设计哲学创建：生成 4-6 段的设计理念文档
- 视觉表达：将哲学转化为画布上的艺术作品
- 设计原则：
  - 强调视觉表达而非文本
  - 最小化文本使用（仅作为视觉元素）
  - 空间、形式、颜色、构图的表达
- 字体资源：提供 canvas-fonts 目录下的多种字体
- 多页支持：可创建多页 PDF 或多个 PNG 文件
- 质量要求：强调专业级工艺，要求作品看起来经过精心制作

**设计理念**: 创建艺术对象而非装饰性文档，强调视觉沟通而非文字解释

## 4. doc-coauthoring (文档协作编写)

**作用**: 提供结构化工作流程，指导用户协作编写文档

**核心功能**:

- 三阶段工作流程：
  1. **上下文收集**：用户提供所有相关上下文，Claude 提出澄清问题
  2. **精炼与结构**：通过头脑风暴和编辑迭代构建每个部分
  3. **读者测试**：使用新的 Claude 实例（无上下文）测试文档，发现盲点
- 文档类型支持：技术规范、决策文档、提案、RFC 等
- 上下文管理：
  - 支持从团队频道、共享文档获取上下文
  - 支持集成工具（Slack、Teams、Google Drive 等）
- 迭代精炼：每个部分通过 5-20 个选项的头脑风暴，用户选择保留/删除/合并
- 质量检查：在接近完成时检查流程、一致性、冗余和矛盾

**优势**: 确保文档在他人阅读时（包括粘贴到 Claude）能正常工作

## 5. docx (Word 文档处理)

**作用**: 全面的 Word 文档创建、编辑和分析，支持跟踪更改、评论、格式保留和文本提取

**核心功能**:

- **创建新文档**：使用 docx-js（JavaScript/TypeScript）创建 Word 文档
- **编辑现有文档**：使用 Document 库（Python）进行 OOXML 操作
- **跟踪更改工作流**（Redlining）：
  - 将文档转换为 markdown 以识别更改
  - 批量实现更改（3-10 个相关更改为一组）
  - 最小化精确编辑：只标记实际更改的文本
- **文本提取**：使用 pandoc 转换为 markdown
- **原始 XML 访问**：用于评论、复杂格式、文档结构、嵌入媒体
- **文档转换**：支持 DOCX 转 PDF，PDF 转图像
- **工具选择**：
  - 简单更改：基本 OOXML 编辑
  - 他人文档/法律/学术/商业文档：必须使用跟踪更改工作流

**技术栈**: docx-js（创建）、Document 库（编辑）、pandoc（文本提取）、OOXML（底层操作）

## 6. frontend-design (前端设计)

**作用**: 创建独特、生产级的前端界面，具有高质量设计，避免通用 AI 美学

**核心功能**:

- **设计思维**：
  - 理解上下文并承诺大胆的美学方向
  - 选择极端风格：极简、极繁、复古未来、有机/自然、奢华/精致等
- **前端美学指南**：
  - **字体**：选择美丽、独特、有趣的字体，避免通用字体（Arial、Inter）
  - **颜色与主题**：使用 CSS 变量保持一致性，主导色配尖锐强调色
  - **动效**：使用动画和微交互，CSS-only 解决方案，Motion 库（React）
  - **空间构图**：意外布局、不对称、重叠、对角线流动、打破网格的元素
  - **背景与视觉细节**：渐变网格、噪声纹理、几何图案、分层透明度等
- **避免通用 AI 美学**：
  - 避免过度使用的字体族（Inter、Roboto、Arial）
  - 避免陈词滥调的颜色方案（特别是紫色渐变白背景）
  - 避免可预测的布局和组件模式

**输出**: 生产级、功能性、视觉冲击力强、具有清晰美学观点的代码

## 8. mcp-builder (MCP 服务器构建器)

**作用**: 创建高质量的 MCP（Model Context Protocol）服务器，使 LLM 能够通过设计良好的工具与外部服务交互

**核心功能**:

- **四阶段工作流程**：
  1. **深度研究与规划**：
     - 理解现代 MCP 设计（API 覆盖 vs 工作流工具）
     - 研究 MCP 协议文档
     - 研究框架文档（TypeScript/Python）
     - 规划实现
  2. **实现**：
     - 设置项目结构
     - 实现核心基础设施（API 客户端、错误处理、响应格式化）
     - 实现工具（输入/输出模式、工具描述、实现、注释）
  3. **审查与测试**：
     - 代码质量审查（DRY 原则、错误处理、类型覆盖）
     - 构建和测试（使用 MCP Inspector）
  4. **创建评估**：
     - 创建 10 个评估问题
     - 确保问题独立、只读、复杂、现实、可验证、稳定
- **推荐技术栈**：
  - 语言：TypeScript（推荐）或 Python
  - 传输：Streamable HTTP（远程服务器）或 stdio（本地服务器）
- **工具设计**：
  - 使用 Zod（TypeScript）或 Pydantic（Python）进行输入验证
  - 定义输出模式（outputSchema）
  - 使用结构化内容（structuredContent）
  - 添加注释（readOnlyHint、destructiveHint 等）

**参考文件**: MCP 最佳实践、TypeScript/Python 实现指南、评估指南

## 9. pdf (PDF 处理)

**作用**: 全面的 PDF 操作工具包，用于提取文本和表格、创建新 PDF、合并/拆分文档和处理表单

**核心功能**:

- **Python 库**：
  - **pypdf**：基本操作（合并、拆分、旋转、元数据提取、密码保护）
  - **pdfplumber**：文本和表格提取（保留布局）
  - **reportlab**：创建新 PDF
- **命令行工具**：
  - **pdftotext**（poppler-utils）：文本提取
  - **qpdf**：合并、拆分、旋转、解密
  - **pdftk**：合并、拆分、旋转（如果可用）
- **常见任务**：
  - 从扫描 PDF 提取文本（OCR，使用 pytesseract）
  - 添加水印
  - 提取图像（pdfimages）
  - 密码保护
- **表单处理**：专门的 forms.md 文件提供表单填写指南
- **高级功能**：参考 reference.md 获取高级 pypdfium2 用法和 JavaScript 库（pdf-lib）

**快速参考**: 根据任务选择最佳工具（pypdf、pdfplumber、reportlab、qpdf 等）

## 10. pptx (PowerPoint 演示文稿处理)

**作用**: 演示文稿创建、编辑和分析，支持布局、评论和演讲者备注

**核心功能**:

- **创建新演示文稿**（无模板）：
  - 使用 html2pptx 工作流将 HTML 幻灯片转换为 PowerPoint
  - 设计原则：内容驱动的设计选择、仅使用 Web 安全字体
  - 颜色调色板：15+ 预设调色板选项
  - 视觉细节选项：几何图案、边框处理、字体处理、图表样式、布局创新
  - 布局提示：两列布局（首选）用于图表/表格
- **创建新演示文稿**（使用模板）：
  - 提取模板文本和创建缩略图网格
  - 分析模板并保存清单到文件
  - 基于模板清单创建演示文稿大纲
  - 使用 rearrange.py 复制、重新排序和删除幻灯片
  - 使用 inventory.py 提取所有文本
  - 生成替换文本并保存到 JSON
  - 使用 replace.py 应用替换
- **编辑现有演示文稿**：
  - 使用 OOXML 格式工作
  - 解包、编辑 XML、验证、打包
- **文本提取**：使用 markitdown 转换为 markdown
- **原始 XML 访问**：用于评论、演讲者备注、幻灯片布局、动画、设计元素
- **缩略图网格**：创建视觉缩略图网格用于快速分析
- **转换为图像**：PPTX → PDF → JPEG 图像

**技术栈**: html2pptx.js、PptxGenJS、markitdown、LibreOffice、Poppler

## 11. skill-creator (Skill 创建工具)

**作用**: 创建有效 skill 的指南，扩展 Claude 的能力，提供专业知识、工作流或工具集成

**核心功能**:

- **Skill 结构**：
  - SKILL.md（必需）：包含 YAML 前置元数据和 Markdown 说明
  - 脚本（scripts/）：可执行代码，用于需要确定性可靠性的任务
  - 参考（references/）：文档和参考材料，按需加载到上下文
  - 资产（assets/）：用于输出的文件（模板、图标、字体等）
- **核心原则**：
  - **简洁是关键**：上下文窗口是公共资源，只添加 Claude 没有的信息
  - **设置适当的自由度**：匹配任务的脆弱性和可变性
  - **渐进式披露**：三级加载系统（元数据、SKILL.md 主体、捆绑资源）
- **Skill 创建流程**：
  1. 用具体示例理解 skill
  2. 规划可重用的 skill 内容
  3. 初始化 skill（运行 init_skill.py）
  4. 编辑 skill（实现资源和编写 SKILL.md）
  5. 打包 skill（运行 package_skill.py）
  6. 基于实际使用迭代
- **编写指南**：
  - 始终使用命令式/不定式形式
  - 前置元数据：name 和 description（description 是主要触发机制）
  - 主体：使用说明和捆绑资源指南

**参考文件**: workflows.md（多步骤流程）、output-patterns.md（输出格式或质量标准）

## 12. slack-gif-creator (Slack GIF 创建器)

**作用**: 创建针对 Slack 优化的动画 GIF 的知识和工具

**核心功能**:

- **Slack 要求**：
  - 尺寸：表情符号 GIF 128x128（推荐），消息 GIF 480x480
  - 参数：FPS 10-30，颜色 48-128，持续时间 3 秒以下（表情符号）
- **核心工作流**：
  1. 创建 GIFBuilder
  2. 生成帧（使用 PIL 原语绘制）
  3. 使用优化保存
- **绘图图形**：
  - 处理用户上传的图像
  - 从零开始绘制（使用 PIL ImageDraw 原语：椭圆、多边形、线条、矩形）
  - 使图形看起来更好：更粗的线条、视觉深度、更有趣的形状、注意颜色
- **可用工具**：
  - GIFBuilder：组装帧并优化 Slack
  - 验证器：检查 GIF 是否符合 Slack 要求
  - 缓动函数：平滑运动而非线性
  - 帧助手：空白帧、渐变背景、绘制圆形、文本、星星
- **动画概念**：
  - 摇动/振动、脉冲/心跳、弹跳、旋转、淡入/淡出、滑动、缩放、爆炸/粒子爆发
- **优化策略**：更少的帧、更少的颜色、更小的尺寸、删除重复、表情符号模式

**依赖**: pillow、imageio、numpy

## 15. webapp-testing (Web 应用测试)

**作用**: 使用 Playwright 与本地 Web 应用程序交互和测试的工具包

**核心功能**:

- **功能支持**：
  - 验证前端功能
  - 调试 UI 行为
  - 捕获浏览器截图
  - 查看浏览器日志
- **决策树**：
  - 静态 HTML：直接读取 HTML 文件识别选择器
  - 动态 Web 应用：服务器是否已运行？
    - 否：使用 `with_server.py` 助手 + 编写简化的 Playwright 脚本
    - 是：侦察然后行动模式
- **可用助手脚本**：
  - `with_server.py` - 管理服务器生命周期（支持多个服务器）
- **侦察然后行动模式**：
  1. 导航并等待 networkidle
  2. 截图或检查 DOM
  3. 从渲染状态识别选择器
  4. 使用发现的选择器执行操作
- **最佳实践**：
  - 使用捆绑脚本作为黑盒
  - 使用 `sync_playwright()` 进行同步脚本
  - 完成后始终关闭浏览器
  - 使用描述性选择器
  - 添加适当的等待

**常见陷阱**: 在动态应用上等待 networkidle 之前不要检查 DOM

## 16. xlsx (Excel 电子表格处理)

**作用**: 全面的电子表格创建、编辑和分析，支持公式、格式、数据分析和可视化

**核心功能**:

- **所有 Excel 文件要求**：
  - 零公式错误：每个 Excel 模型必须零错误（#REF!、#DIV/0!、#VALUE!、#N/A、#NAME?）
  - 保留现有模板：修改文件时完全匹配现有格式、样式和约定
- **财务模型标准**：
  - **颜色编码**：
    - 蓝色文本：硬编码输入和用户将更改的数字
    - 黑色文本：所有公式和计算
    - 绿色文本：从同一工作簿中其他工作表拉取的链接
    - 红色文本：指向其他文件的外部链接
    - 黄色背景：需要关注的关键假设或需要更新的单元格
  - **数字格式**：年份、货币、零、百分比、倍数、负数
  - **公式构造规则**：假设放置、公式错误预防、硬编码文档要求
- **读取和分析数据**：
  - 使用 pandas 进行数据分析、可视化和基本操作
- **Excel 文件工作流**：
